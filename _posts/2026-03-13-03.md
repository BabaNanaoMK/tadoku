## Antigravity の概要

<pre style="background:#252525;color:#f3f4f6;border:1px solid #505050;border-radius:12px;padding:16px 18px;overflow-x:auto;line-height:1.7;font-size:0.95rem;white-space:pre;">
Google Antigravity
├─ 1. 何をする道具か
│  ├─ 1.1 AIエージェント中心の開発環境 
│  │  ├─ 人が1行ずつコードを書く代わりに、まとまった仕事をAIに任せる  
│  │  ├─ 単なるコード補完より「計画→実装→確認」が強い
│  │  └─ ソフトウェア開発を主目的にした道具
│  ├─ 1.2 位置づけ
│  │  ├─ VS Code系に近い操作感の開発環境
│  │  ├─ AIチャットツール単体ではなく、作業実行まで踏み込む
│  │  └─ 初心者の“相談相手”にも、開発者の“作業代行”にも寄る
│  └─ 1.3 得意な場面
│     ├─ 新機能の追加
│     ├─ バグ修正
│     ├─ テスト作成
│     ├─ 動作確認
│     └─ 反復作業の自動化
│
├─ 2. 中核の考え方
│  ├─ 2.1 Agent-first
│  │  ├─ 人が主役でAIが補助、ではなく
│  │  ├─ AIがまとまった単位で仕事を進め
│  │  └─ 人がレビューと方向修正をする
│  ├─ 2.2 Multi-step reasoning
│  │  ├─ 1発回答ではなく複数段階で考える
│  │  ├─ 必要ならコードベースを読み
│  │  ├─ 実装し
│  │  └─ 検証までつなぐ
│  └─ 2.3 Verifiable work
│     ├─ 「やりました」だけでなく
│     ├─ 計画
│     ├─ タスクリスト
│     ├─ スクリーンショット
│     └─ ブラウザ記録などで確認しやすくする
│
├─ 3. ユーザーが触る主な画面
│  ├─ 3.1 Editor view
│  │  ├─ ふつうのIDEに近い画面
│  │  ├─ コードを読む・編集する
│  │  ├─ エージェントにその場で依頼する
│  │  └─ 実装結果を確認する
│  ├─ 3.2 Manager view
│  │  ├─ 複数エージェントの管理画面
│  │  ├─ どの仕事が進行中かを見る
│  │  ├─ 並列で別タスクを走らせる
│  │  └─ いわば「司令塔」「ミッションコントロール」
│  └─ 3.3 Side panel / Agent panel
│     ├─ エージェントとの会話
│     ├─ 指示の投入
│     ├─ 途中成果の確認
│     └─ 修正依頼・再実行
│
├─ 4. エージェントが使える作業手段
│  ├─ 4.1 Editor access
│  │  ├─ 既存コードを読む
│  │  ├─ 複数ファイルを編集する
│  │  └─ 差分を作る
│  ├─ 4.2 Terminal access
│  │  ├─ コマンド実行
│  │  ├─ ビルド
│  │  ├─ テスト
│  │  └─ ローカル起動
│  ├─ 4.3 Browser access
│  │  ├─ Web画面を開く
│  │  ├─ UI動作を確認する
│  │  ├─ 必要に応じてブラウザ操作する
│  │  └─ 画面確認を証跡として残す
│  └─ 4.4 Context access
│     ├─ プロジェクト全体の文脈を読む
│     ├─ 過去の作業を参照する
│     └─ 依頼内容に沿って必要箇所を探す
│
├─ 5. AIに任せられる代表作業
│  ├─ 5.1 計画づくり
│  │  ├─ 要件整理
│  │  ├─ 実装手順の提案
│  │  └─ タスク分解
│  ├─ 5.2 実装
│  │  ├─ 新規コード作成
│  │  ├─ 既存コード修正
│  │  ├─ リファクタリング
│  │  └─ 設定ファイル更新
│  ├─ 5.3 検証
│  │  ├─ テスト実行
│  │  ├─ ブラウザ確認
│  │  ├─ エラー再現
│  │  └─ 修正後チェック
│  ├─ 5.4 説明
│  │  ├─ このコードの意味は何か
│  │  ├─ どこを変えたか
│  │  ├─ なぜその修正にしたか
│  │  └─ 次に何を確認すべきか
│  └─ 5.5 繰り返し改善
│     ├─ フィードバックを受けて再修正
│     ├─ 方針変更に追従
│     └─ 小さく試して直す
│
├─ 6. 成果物として見えるもの
│  ├─ 6.1 Artifacts
│  │  ├─ タスクリスト
│  │  ├─ 実装計画
│  │  ├─ スクリーンショット
│  │  ├─ ブラウザ録画
│  │  └─ 作業の証跡
│  ├─ 6.2 Code changes
│  │  ├─ 変更ファイル
│  │  ├─ 差分
│  │  └─ テスト追加や修正
│  └─ 6.3 Review point
│     ├─ 人が確認すべき箇所
│     ├─ 残タスク
│     └─ リスクや注意点
│
├─ 7. 使い方の流れ
│  ├─ 7.1 依頼する
│  │  ├─ 何を作るか
│  │  ├─ 何を直すか
│  │  └─ 制約は何か
│  ├─ 7.2 エージェントが調査する
│  │  ├─ コードを読む
│  │  ├─ 関係ファイルを探す
│  │  └─ 実行方法を確認する
│  ├─ 7.3 計画を立てる
│  │  ├─ 手順化
│  │  ├─ 必要作業の分解
│  │  └─ 確認ポイントの整理
│  ├─ 7.4 実装する
│  │  ├─ ファイル編集
│  │  ├─ コマンド実行
│  │  └─ 必要ならブラウザ確認
│  ├─ 7.5 結果を示す
│  │  ├─ 変更内容
│  │  ├─ 証跡
│  │  └─ 次のアクション
│  └─ 7.6 人が採用・修正指示する
│     ├─ 受け入れ
│     ├─ 追加修正
│     └─ 差し戻し
│
├─ 8. 複数エージェント運用
│  ├─ 8.1 並列作業
│  │  ├─ 別機能を同時に進める
│  │  ├─ 調査担当と実装担当を分ける
│  │  └─ 比較案を複数作る
│  ├─ 8.2 管理
│  │  ├─ 進捗監視
│  │  ├─ 役割分担
│  │  └─ 成果比較
│  └─ 8.3 向いているケース
│     ├─ 大きめのコードベース
│     ├─ 複数の課題がある時
│     └─ 試行錯誤を早めたい時
│
├─ 9. 外部サービスや拡張
│  ├─ 9.1 Google Cloud 連携
│  │  ├─ アプリ構築
│  │  ├─ デプロイ
│  │  └─ クラウド資源との接続
│  ├─ 9.2 Data Cloud extension
│  │  ├─ BigQuery などのデータ資源をIDEから扱う
│  │  ├─ データ分析・データエンジニア向け用途
│  │  └─ Antigravity/VS Code 用拡張として提供
│  └─ 9.3 モデル利用の広がり
│     ├─ Gemini中心
│     └─ 一部報道では他モデル対応にも言及あり
│
├─ 10. 初心者が理解しておくべき制約
│  ├─ 10.1 万能ではない
│  │  ├─ 指示が雑だと出力も雑になりやすい
│  │  ├─ 間違った修正をすることはある
│  │  └─ 人のレビューは必要
│  ├─ 10.2 開発向けの道具
│  │  ├─ 文章専用ツールではない
│  │  ├─ 主戦場はコード・設定・検証
│  │  └─ 非エンジニアでも使えるが、開発文脈が強い
│  └─ 10.3 信頼の作り方が重要
│     ├─ 何を変えたか確認する
│     ├─ テスト結果を見る
│     └─ Artifactsで裏取りする
│
└─ 11. ひとことで言うと
   ├─ 11.1 従来型
   │  └─ 「コードを書く時に横で手伝うAI」
   └─ 11.2 Antigravity型
      └─ 「仕事のかたまりを渡すと、調べて・作って・確かめるAI開発環境」
</pre>
